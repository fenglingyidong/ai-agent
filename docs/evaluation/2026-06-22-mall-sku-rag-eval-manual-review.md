## 优先问题

1. `mall_search_products` 对商品名或别名召回不稳，Q05 彩色中性笔被判无匹配。
2. 复杂组合题的候选池召回偏窄，Q15、Q18、Q29、Q30 都漏掉关键品类。
3. 组合采购缺少“按子需求拆多次检索”的策略，容易只围绕第一批候选回答。
4. 对比题需要避免把待比较商品属性当作用户硬约束，Q20 因此结论偏窄。

## 建议

优先改进 RAG 组合检索策略：对包含多个品类或多个用途的需求，先从路由或 prompt 中拆出子需求，再分别调用 `searchProductKnowledge`，最后合并候选并做预算计算。其次补强商城搜索的名称规范化和别名匹配，避免简单事实题因搜索词差异直接漏召回。
